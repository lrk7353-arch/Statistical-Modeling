## 17. 用户质疑“潜在实现路径”太虚，转向沉睡资产切口

用户指出：

- “潜在实现路径”很虚，不够靠谱。
- 公共数据沉睡资产识别与激活路径推荐似乎更实在：
  - 哪些数据开放了但没人用？
  - 为什么没人用？
  - 怎么组合后变得有价值？
- 但继续质疑：
  - 沉睡资产是不是太好判别了？
  - 不就是下载量/调用量少吗？
  - 激活路径怎么体现？

助手回应：

- 如果定义成“下载量低 = 沉睡资产”，确实肤浅。
- 应把“沉睡资产”定义成一个**反事实问题**：

> 在数据质量、治理场景相关性和可组合价值都较高的情况下，实际使用表现显著低于模型预期的数据资源，才叫沉睡资产。

换言之：

> 不是没人下载就沉睡，而是本来应该有人用，但实际没人用。

步骤：

1. 预测“应得使用量”：
   - 用数据集质量、主题、格式、更新频率、字段说明、API、空间/时间序列属性、语义清晰度、所属部门、同主题平均热度、核心场景等特征预测理论使用量。

2. 计算“使用残差”：
   - `沉睡度 = 预测使用量 - 实际使用量`
   - 预测高、实际低，才是高潜低用资源。

3. 高潜低用识别：
   - 预测热度 Top 30%，实际热度 Bottom 50%。
   - 或标准化残差显著为负。
   - 或预期转化概率高但无应用关联。

解释“为什么没人用”：

- 可用性障碍：格式差、无 API、字段说明弱。
- 时效性障碍：更新慢。
- 语义障碍：标题摘要不清楚。
- 组合障碍：单个数据集价值低，缺少组合推荐。
- 场景障碍：没有绑定具体治理场景。
- 供需错配：部门开放方向与用户需求不一致。

沉睡资产分型：

- 高质量低曝光型。
- 可组合但孤岛型。
- 关键场景缺 API 型。
- 说明不清型。
- 更新滞后型。
- 主题冷门型。

激活路径：

- 不是泛泛宣传，而是可模拟策略：
  - 增加 API。
  - 提高更新频率。
  - 补充字段说明。
  - 关联核心应用场景。
  - 推荐组合数据包。
  - 转换为机器可读格式。
  - 加入专题数据包。
  - 作为典型应用数据源。

两种实现：

1. 反事实模拟：
   - 例如模拟“如果提供 API”，转化概率从 0.15 提升到 0.28。

2. 相似成功案例匹配：
   - 找相似但高使用的数据集 B，对比沉睡数据集 A。
   - 如果 B 有 API、应用场景清晰、摘要清楚、组合关系多，则 A 的激活路径可参考 B。

组合变得有价值：

- 单独公交站点数据一般。
- 加上老龄人口、社区卫生服务中心、养老机构，可做老年就医可达性。
- 加上道路、事故点、学校，可做校园周边交通安全。
- 加上商圈、夜间灯光、地铁末班车，可做夜间经济服务保障。

硬伤：

- 没有真实干预实验，推荐策略效果不一定能验证。
- 可通过以下方式补强：
  - 历史版本对比。
  - 平台版本日志冲击。
  - 类似数据集对照。
  - 小规模用户实验。

建议题名：

> **公共数据“沉睡资产”的识别与激活路径推荐研究——基于使用残差、语义组合与反事实模拟**

或：

> **公共数据高潜低用资源识别与场景化激活推荐研究——以上海公共数据开放平台为例**
